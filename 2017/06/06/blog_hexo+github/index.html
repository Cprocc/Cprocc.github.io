hexo+github搭建博客

摘要
使用github+hexo搭建个人博客

正文：

终于完成搭建博客，现在分享自己的历程

准备工作

  1. 注册github，建立名为 username.github.io仓库
  2. 下载安装node.js 官网：https://nodejs.org/en/download/ 起初下载很慢，或者资源丢失，搬了梯子后正常。此后的所有下载都在有梯子的条件下

  3. 下载安装git 官网：https://git-scm.com/download/win

    如图表示成功

  4. 安装hexo：桌面右键 git bash here：输入命令

    npm install -g hexo-cli

    开始博客

    hexo命令介绍

  • hexo generate (hexo g) 生成静态文件，会在当前目录下生成一个新的叫做public的文件夹
  • hexo server (hexo s) 启动本地web服务，用于博客的预览
  • hexo deploy (hexo d) 部署博客到远端服务器
  • hexo new “postName” #新建文章
  • hexo new page “pageName” #新建页面
  1. 本地建立文件夹，我的选择 E:\my_blog
  2. 初始化 选中my_blog文件夹，右键git bash here，进入。输入
    hexo init

    系统会进入下载状态，下载hexo 提供的一套初始界面，完成后可以手动查看原来的my_blog文件夹里多了一堆相关的文件
  3. npm install自动安装相关插件
  4. 输入hexo g生成文件 ， 输入hexo s启动本地预览，打开浏览器输入http://localhost:4000/ 预览博客，键入ctrl+c关闭本地web服务

部署到github仓库

  1. 配置git 检查ssh密匙，第一次使用Git应该是本地不存在ssh密匙的，需要新建
  2. 生成ssh密匙： ssh-keygen -t rsa -C "注册github时的邮箱"会提示键入文件位置，和输入密码，确认密码。不需要，三次回车即可
  3. 获取本地Git的key：cat ~/.ssh/id_rsa.pub
  4. 复制返回的key，添加到github中。new ssh key 成功，一般github会向你的github认证邮箱中发一条邮件
  5. 配置本地的my_blog文件夹中的_config.yml内容，等于将github地址添加为本地hexo的部署目标
    在_config.yml文档末尾添加
    deploy:
    type: git
    repo: ssh://git@github.com/username/username.github.io
    branch: master

  6. npm install hexo-deployer-git –save

  7. 右键my_blog git bash here, hexo g生成静待界面 hexo s本地服务器打开进行测试 hexo d部署到git

  8. 跳出github 登录界面：输入账户密码即可
