python高级编程

python循环导入

  • a调用b，b调用a
  • 怎样避免
    1. 程序设计上分层，降低耦合
    2. 导入语句放在需要导入的函数体内

copy&deepcopy

  • import copy
  • copy.copy() 浅拷贝，只拷贝父类元素，且对于不可变元素的拷贝变成简单的赋值操作如：tuple
  • copy.deepcopy() 深拷贝

进制&位运算

  • 计算机中，数字是二进制补码的形式存储的
  • 正数：原码==反码==补码
  • 负数：反码==符号为不变，其他位取反，补码==反码+1
    • eg：~9 = -10

类和变量的私有化

  • XX公有变量/方法
  • __XX:避免与子类中的属性明明冲突，无法在外部直接访问（由于名字重整name mangling）
  • XX magic方法
  • _x :单前置下划线，私有化属性或方法。from someModule import*禁止导入，但是类对象和子类可以访问
  • X_:单后置下划线，用于避免与python关键字冲突
  • from privat import* 无法导入下划线的名字方法
  • import privat 可以导入下划线的名字方法

staticmethod 和 classmethod的区别

垃圾回收

  1. 小正数对象池 [-5,257]
  2. 大正数对象池
  3. intern 机制
  4. GC
    • 无法解决循环引用的问题。n1.next = n2 n2.prev = n1,当n1，n2都置为空时也计数器仍为1
    • Generation Zero，Python使用一种不同的活跃的链表来保持追踪活跃的对象。python会循环遍历零代列表上的每个对象，根据规则减掉其引用计数，在这个过程中，python还会一个接一个的统计内部引用量以防止过早的释放现象。

  • 类也是一种对象
  • 输出class，class()多了一行class object at…..
  • type(class) is type
  • type不仅可以用来查看类型，也可以用来创建类

元类

生成器

  • a = [x*2 for x in range(10000000)]//可能崩溃
  • 列表元素按照某种算法推出来，在循环过程中不断推算出后续的元素，这样就不必创建完整的list，从而节省大量的空间。这种机制在python中成为生成器:generator
  • 生成器是可迭代对象
  • 生成一个简单的迭代器G = (x for x in range(5))，next(G)逐个访问
  • g1 = G这里用的是浅拷贝
  • fib(5)调用时返回的是一个对象,a = fib(5),next(a) // a.next(),调用第一次的时候只执行到yield这个语句。什么时候调用就什么时候执行一次生成器循环,以异常结束。用for num in range(a)，正常结束。
  • yield,是一个类似于return的关键字，遇到yield时就返回yield后面的值。重点是：下一次迭代时，从上一次迭代遇到的yield后面的代码开始执行
  • send(msg)与next()的区别:当出现temp = yield i这样的语句时，一次next temp获取不到值，利用send(msg),msg将会给到temp。但是第一次对迭代器进行操作就调用send()会出现错误,可以send(None)。

迭代器

  • 迭代是访问集合的一种方式，迭代器是一个可以记住遍历的位置的对象。迭代器对象从集合的第一个元素开始访问，直到访问结束。
  • 可迭代对象
    • 数据结合，list，tuple，dict，set，str
    • generator，包括生成器和带yiled的generator function
    • 统称为iterable
  • from collections import Iterable， isinstance([],Iterable)
  • 迭代器
    • from collections import Iterator,isinstance([],Iterator)
    • 可迭代对象不一定是迭代器
    • 迭代器可使用next()方法
    • iter(),用来转化。isinstance(iter([]),Iterator)

闭包

输出“–1–”

  • 什么是闭包：在函数内部定义一个函数，并且这个函数用到了本身外部的变量
  • 在大函数内部返回闭包，在外部可以通过获取函数返回值调用闭包，外部函数的结果暂时存储在栈中
  • 闭包的应用：构建线性方程，a*x + b简化调用过程，一种函数产生多种模式同时存在调用。

装饰器

  1. 闭包的作用之一:装饰器
  2. 装饰器的调用过程和@顺序相同，和装饰器顺序相反
  3. 在python解释器开始工作的时候，装饰器就开始进行装饰了，不需要等到调用
  4. 有参数/不定长参数/带有返回值的类型进行装饰
